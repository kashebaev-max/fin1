# 📸 Генератор скриншотов для справочного центра Finstat.kz

Автоматический скрипт на **Playwright**, который заходит в систему и делает все необходимые скриншоты для инструкции пользователя.

## 🎯 Что делает

1. Запускает браузер
2. Логинится в вашу систему через тестовый аккаунт
3. Проходит по всем основным модулям
4. Делает ~115 скриншотов с правильными именами
5. Сохраняет в правильные папки в `public/help/`

## ⚙ Подготовка

### 1. Создайте тестовый аккаунт

В системе Finstat создайте отдельный аккаунт **только для скриншотов**:

- Email: `screenshots@finstat.kz` (или любой другой)
- Пароль: придумайте надёжный
- Заполните **полный профиль компании** (все поля)
- Добавьте **минимум данных** для красивых скриншотов:
  - 3-5 контрагентов
  - 2-3 заказа
  - 5-10 товаров/услуг
  - 2-3 сотрудника
  - Несколько проводок (можно через AI Жанару)

> 💡 **Совет:** Используйте тестовые данные с реалистичными названиями. Например, контрагент «ТОО Альфа-Сервис», сумма 580 000 ₸, и т.д.

### 2. Установите зависимости

В терминале **Codespace** или локально:

```bash
cd .scripts/screenshots
npm install
npx playwright install chromium
```

Это займёт 1-2 минуты.

## 🚀 Запуск

### Вариант A: Через переменные окружения (рекомендуется)

```bash
export FINSTAT_URL=https://finstat.kz
export TEST_EMAIL=screenshots@finstat.kz
export TEST_PASSWORD=YourPassword123

cd .scripts/screenshots
node generate-screenshots.js
```

### Вариант B: Прямо в файле

Откройте `generate-screenshots.js` и исправьте константы вверху:

```javascript
const CONFIG = {
  BASE_URL: "https://finstat.kz",
  TEST_EMAIL: "screenshots@finstat.kz",
  TEST_PASSWORD: "YourPassword123",
  // ...
};
```

Потом запустите:

```bash
node generate-screenshots.js
```

## ⚠ Если что-то идёт не так

### Скрипт не может залогиниться

**Причина 1:** Неправильный email/пароль

Проверьте `TEST_EMAIL` и `TEST_PASSWORD` в конфиге.

**Причина 2:** Селекторы изменились

Если форма входа изменилась, отредактируйте функцию `login()` в `generate-screenshots.js`:

```javascript
// Найти эти строки и исправить селекторы:
await safeFill(page, 'input[type="email"]', CONFIG.TEST_EMAIL);
await safeFill(page, 'input[type="password"]', CONFIG.TEST_PASSWORD);
```

### Какие-то скриншоты пропускаются

Это **нормально** — скрипт умеет пропускать недоступные модули.

В консоли увидите:
```
⚠ Не удалось кликнуть button:has-text("..."): timeout
```

Эти скриншоты можно сделать **руками** позже.

### Скриншоты получились пустыми/неправильными

Откройте `generate-screenshots.js` и:

1. Поставьте `HEADLESS: false` (увидите что делает скрипт)
2. Поставьте `SLOW_MO: 1000` (замедлит действия в 10 раз)
3. Запустите снова и наблюдайте

## 🎨 Кастомизация

### Изменить размер скриншотов

```javascript
VIEWPORT: { width: 1920, height: 1080 }  // Full HD
```

### Изменить тему

```javascript
THEME: "dark"  // или "light"
```

### Пропустить какие-то разделы

```javascript
SKIP_SECTIONS: ["15-sono-submit", "18-migration"]
```

### Запустить в headless режиме (без UI)

```bash
HEADLESS=true node generate-screenshots.js
```

Полезно для **CI/CD** или серверов.

## 📤 Загрузка результатов

После успешной генерации:

```bash
# Из корня проекта
git add public/help/
git commit -m "feat: автогенерированные скриншоты для help"
git push
```

Netlify автоматически задеплоит изменения, и скриншоты появятся в `/dashboard/help`.

## 🔄 Перегенерация

Если интерфейс изменился — просто запустите скрипт ещё раз:

```bash
node generate-screenshots.js
```

Старые скриншоты перезапишутся новыми. Никаких ручных действий не требуется.

## 💡 Советы

1. **Запускайте при первой настройке** — после Pack 59 пакета и до запуска продаж
2. **Перегенерируйте после крупных обновлений** UI — раз в месяц
3. **Используйте отдельный тестовый аккаунт** — не свой основной
4. **Проверяйте каждый скриншот** — какие-то всё равно нужно будет переснять руками
5. **Делайте бэкап** старых скриншотов перед перегенерацией

## 🐛 Если совсем не работает

Альтернативные подходы:

1. **Делайте скриншоты руками** по списку из `docs/SCREENSHOTS-CHECKLIST.md`
2. **Запишите видео** через Loom — даже эффективнее
3. **Используйте SVG-иллюстрации** вместо скриншотов
