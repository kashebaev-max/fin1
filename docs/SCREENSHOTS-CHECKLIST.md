# 📸 Список скриншотов для справочного центра

Этот документ — **руководство по созданию скриншотов** для инструкции пользователя.

## 📁 Куда складывать

Все скриншоты должны быть в папке **`public/help/`** проекта. Создайте структуру:

```
public/
└── help/
    ├── 01-registration/
    ├── 02-first-setup/
    ├── 03-interface/
    ├── 04-profile/
    ├── 05-counterparties/
    ├── 06-orders/
    ├── 07-nomenclature/
    ├── 08-stock/
    ├── 09-entries/
    ├── 10-osv/
    ├── 11-employees/
    ├── 12-payroll/
    ├── 13-fno/
    ├── 14-sono-create/
    ├── 15-sono-submit/
    ├── 16-zhanara/
    ├── 17-actions/
    ├── 18-migration/
    └── 19-exports/
```

## 🎨 Технические требования

- **Формат:** PNG (предпочтительно) или JPG
- **Размер:** ширина **1200-1600px** (для retina-экранов)
- **Соотношение сторон:** 16:9 (рекомендуется) — система уже подготовлена под него
- **Размер файла:** до **500KB** (после оптимизации)
- **Имя файла:** строго по списку ниже (с английскими буквами и цифрами)

## 💡 Подсказки по созданию скриншотов

1. **Используйте тестовые данные** — не реальных клиентов и не реальные суммы
2. **Скрывайте email и БИН** — заменяйте на тестовые (например, `test@finstat.kz`, БИН `123456789012`)
3. **Подсветите важное** — стрелками или прямоугольниками (можно в Mac Preview, Snipaste, Lightshot, ShareX)
4. **Светлая тема** для большинства скриншотов — выглядит чище и понятнее в инструкции
5. **Полный экран браузера** — без вкладок и адресной строки если возможно

## 🚀 Как загрузить в систему

### Вариант 1: Прямо в репозиторий (рекомендуется)

1. Сделайте все скриншоты по списку выше
2. Положите их в правильные папки в `public/help/`
3. Закоммитьте в репозиторий:
   ```bash
   git add public/help/
   git commit -m "feat: добавлены скриншоты для справочного центра"
   git push
   ```
4. Netlify автоматически задеплоит — скриншоты появятся в инструкции

### Вариант 2: Загрузить в Supabase Storage

1. В Supabase создайте bucket `help-screenshots` (публичный)
2. Загрузите файлы через интерфейс Supabase
3. Скопируйте public URLs
4. В файле `lib/help-content.ts` замените пути с `/help/...` на полные URLs

### Тестирование

После добавления скриншота — обновите страницу `/dashboard/help` и проверьте что он отображается.

Если скриншот не отображается:
- Проверьте имя файла (точное совпадение с тем что в `help-content.ts`)
- Проверьте что папка `public/help/` создана и закоммичена
- Проверьте что файл < 500KB (большие изображения могут не загрузиться)

## 💡 Альтернатива: автоматизация скриншотов

Если хочешь делать скриншоты массово — можно использовать инструменты:

- **Playwright** или **Puppeteer** — автоматическая генерация скриншотов через скрипт
- **Loom** — короткие видео-инструкции (можно вставить вместо скриншотов)
- **Tella.tv** — браузерная запись с акцентом на курсоре

Если интересует — могу написать скрипт автоматической генерации скриншотов на Playwright.
